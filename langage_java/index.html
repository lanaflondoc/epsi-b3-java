Exercices¶

Exercice - Programme simple

Testez le programme Java suivant :

public class Programme {

    public static void main(String[] args) {
        System.out.println("Bonjour");
    }

}

Pour fonctionner, ce programme doit ce trouver dans un fichier Programme.java dans le répertoire des sources du projet.

System.out représente la sortie standard du programme.

Exercice - Demander son nom

Écrivez un programme qui demande à l’utilisateur son nom et qui l’affiche. Pour cela, il faut utiliser la classe Scanner :

import java.util.Scanner;

public class ProgrammeSaisie {

        public static void main(String[] args) {
                Scanner sc = new Scanner(System.in);

                String entreeSaisie = sc.nextLine();
        }

}

Exercice - Calculer

Écrivez un programme qui demande une série de 10 nombres à l’utilisateur. Le programme affiche ensuite :

  • Le nombre le plus grand

  • Le nombre le plus petit

  • La somme des nombres

  • La moyenne des nombres

  • Le nombre de nombres pairs

Astuce

La classe Scanner peut lire directement un nombre à partir des données saisies par l’utilisation. Sinon vous pouvez utiliser Double.parseDouble

double nombre = Double.parseDouble("1.25");

Exercice - La classe Ascenseur

On représente un ascenseur dans un système à l’aide d’une classe. La classe Ascenseur doit disposer de la méthode publique getEtage() qui retourne le numéro de l’étage auquel se trouve l’ascenseur. Si l’ascenseur est au rez-de-chaussée, la méthode retourne 0.

La classe Ascenseur a également la méthode appeler(int etage) qui permet d’appeler l’ascenseur à un étage donné. Cette méthode a pour conséquence d’amener l’ascenseur à l’étage donné.

Implémentez la classe Ascenseur. Ajoutez dans la classe une méthode :

public static void main(String[] args) {
}

Dans la méthode main, créez un objet de type Ascenseur et appelez ses méthodes. Après chaque appel, affichez sur la sortie standard l’étage où se trouve l’ascenseur.

Exercice - La classe BoiteDeVitesses

On souhaite créer une classe BoiteDeVitesses pour représenter une boite de vitesses. Cette classe contient les méthodes :

changerVitesse()

Pour passer à la vitesse supérieure

retrograder()

Pour passer à la vitesse inférieure

getVitesse()

Pour donner la vitesse courante (0 représente le point mort)

Implémentez la classe BoiteDeVitesses. Ajoutez dans la classe une méthode :

public static void main(String[] args) {
}

Dans la méthode main, créez un objet de type BoiteDeVitesses pour créer un programme qui demande à l’utilisateur s’il veut changer de vitesse ou rétrograder. Le programme affiche à chaque fois la vitesse courante de la boite.

Exercice - Détecteur de température

On souhaite implémenter une classe DetecteurTemperature. Cette classe recevra des informations sur la température et sera capable de déduire si le niveau de température est normal, élevé ou critique :

Température

Niveau

t ≤ 30

normal

30 < t < 38

élevé

t ≥ 38

critique

La classe DetecteurTemperature possède les méthodes suivantes :

setTemperature(float t)

Pour positionner la température actuelle.

getNiveau()

Qui retourne le niveau de la température actuelle sous la forme d’une chaîne de caractères : « normal », « élevé », « critique »

isNiveauAtteint(String niveau)

Qui retourne true ou false si le niveau courant est au moins le niveau donné en paramètre.

Ajouter la méthode main suivante pour tester votre classe :

public static void main(String[] args) {
  DetecteurTemperature detecteurTemperature = new DetecteurTemperature();

  for(float temperature = 25; temperature < 45; temperature += 2) {
    detecteurTemperature.setTemperature(temperature);
    System.out.println("Température actuelle    = " + temperature);
    System.out.println("Niveau actuel           = " + detecteurTemperature.getNiveau());
    System.out.println("Niveau normal atteint   = " + detecteurTemperature.isNiveauAtteint("normal"));
    System.out.println("Niveau élevé atteint    = " + detecteurTemperature.isNiveauAtteint("élevé"));
    System.out.println("Niveau critique atteint = " + detecteurTemperature.isNiveauAtteint("critique"));
    System.out.println("###################################");
  }
}

Exercice - La classe BoiteDeVitesses (Version 2)

On souhaite créer une classe BoiteDeVitesses pour représenter une boite de vitesses. Cette classe contient les méthodes :

changerVitesse()

Pour passer à la vitesse supérieure. Il n’est pas possible d’aller au delà de la cinquième vitesse (dans ce cas, un appel à la méthode doit être sans effet).

retrograder()

Pour passer à la vitesse inférieure. Il n’est pas possible de rétrograder en dessous de la première (dans ce cas, un appel à la méthode doit être sans effet).

setPointMort()

Pour aller au point mort.

getVitesse()

Pour donner la vitesse courante (0 représente le point mort).

toString()

Qui retourne une chaîne de caractères (String). Si la vitesse vaut 0, la méthode retourne « point mort ». Pour les autres vitesses, la méthode retourne respectivement : « première », « seconde », « troisième », « quatrième », « cinquième ».

Créez un programme qui demande à l’utilisateur les actions qu’il veut faire : changer de vitesse, rétrograder ou passer au point mort. À chaque choix de l’utilisateur, on affiche la vitesse courante.

Exercice - La classe Phrase

Implémenter la classe Phrase. Cette classe permet de créer une chaîne de caractères en ajoutant des mots qui seront séparés par un séparateur. Par défaut, le séparateur est le caractère espace, mais il peut être changé.

La classe Phrase doit permettre de :

  • ajouter un mot

  • ajouter une série de mots

  • ajouter un mot en le répétant n fois

  • changer le séparateur qui peut être soit une chaîne de caractères soit un seul caractère

  • générer la phrase terminée par un point lorsqu’on demande la représentation sous la forme d’une chaîne de caractères de l’objet

  • connaître le nombre de lettres dans la phrase

Le code suivant :

Phrase phrase = new Phrase();
phrase.ajouter("Une");
phrase.ajouter("classe");
phrase.ajouter("pour");
phrase.ajouter("ajouter");
phrase.ajouter("des mots");
phrase.setSeparateur(" et encore ");
phrase.ajouter("des mots", 3);
phrase.setSeparateur(' ');
phrase.ajouter("toujours", "et", "encore");

System.out.println(phrase);
System.out.println(phrase.getNbLettres());

doit produire sur la sortie standard

Une classe pour ajouter des mots et encore des mots et encore des mots et encore des mots toujours et encore.
88

Indication

On ne peut pas parcourir directement une chaîne de caractères, par contre on peut obtenir un tableau de caractères à partir d’une chaîne avec la méthode toCharArray.

String helloWorld = "Hello world!";
for (char c : helloWorld.toCharArray()) {
  // ...
}

Pour savoir si un caractère est une lettre, on utilise la méthode static Character.isAlphabetic.

Exercice - Les constructeurs pour la classe Phrase

En reprenant la classe Phrase, ajoutez les constructeurs suivants :

// constructeur sans paramètre
Phrase phrase = new Phrase();
phrase.ajouter("Bonjour", "le", "monde");
System.out.println(phrase); // Bonjour le monde.

// constructeur pour indiquer quel caractère fini la phrase
Phrase phraseInterrogative = new Phrase(Phrase.INTERROGATION);
phraseInterrogative.ajouter("Ça", "va");
System.out.println(phraseInterrogative); // Ça  va ?

Phrase phraseExclamative = new Phrase(Phrase.EXCLAMATION);
phraseExclamative.ajouter("Oui");
System.out.println(phraseExclamative); // Oui !

Phrase phraseDeclarative = new Phrase(Phrase.DECLARATION);
phraseDeclarative.ajouter("C'est", "bien");
System.out.println(phraseDeclarative); // C'est bien.

// constructeur pour indiquer quel caractère fini la phrase
// et passer un nombre quelconque de mots à la phrase
phrase = new Phrase(Phrase.INTERROGATION, "Et", "ensuite");
System.out.println(phrase); // Et ensuite ?

Ajoutez également des méthodes de classe pour créer directement une Phrase selon son type :

phrase = Phrase.interrogation("Comment", "allez", "vous");
System.out.println(phrase); // Comment allez vous ?

phrase = Phrase.exclamation("Dormez");
System.out.println(phrase); // Dormez !

phrase = Phrase.declaration("Brian", "est", "dans", "la", "cuisine");
System.out.println(phrase); // Brian est dans la cuisine.

Exercice - Implémenter le code de César

Implémenter le code de César en utilisant le code ci-dessous. Cet algorithme de chiffrement classique, utilisé par Jules César dans ses correspondances secrètes, se base sur le principe du décalage. Une lettre est remplacée par son équivalent dans l’alphabet en effectuant un décalage. Le programme ci-dessous utilise un décalage de 23. Cela donne comme équivalence :

  • a → x

  • b → y

  • c → z

  • d → a

  • e → b

  • f → c

  • g → d

  • h → e

  • i → f

  • j → g

  • k → h

  • l → i

  • m → j

  • n → k

  • o → l

  • p → m

  • q → n

  • r → o

  • s → p

  • t → q

  • u → r

  • v → s

  • w → t

  • x → u

  • y → v

  • z → w

Ainsi la phrase :

In cryptography, a Caesar cipher is one of the simplest and most widely known encryption techniques.

sera encodée avec un décalage de 23 comme ceci :

Fk zovmqldoxmev, x Zxbpxo zfmebo fp lkb lc qeb pfjmibpq xka jlpq tfabiv hkltk bkzovmqflk qbzekfnrbp.

Le programme suivant vérifie que le résultat du chiffrement est bien conforme au ce qui est attendu. Dans ce cas, il affichera true sur la sortie standard.

Décommentez le code de chaque étape dans la méthode main au fur et à mesure que vous implémentez la méthode encrypte afin de vérifier que votre algorithme fonctionne correctement.

Indication

On peut obtenir un tableau de caractères à partir d’une chaîne avec la méthode toCharArray. S’il n’est pas possible de parcourir les éléments d’une chaîne de caractères avec un for amélioré, on peut facilement parcourir le tableau de caractères :

String helloWorld = "Hello world!";
for (char c : helloWorld.toCharArray()) {
  // ...
}

Il est possible de créer une nouvelle chaîne de caractères à partir d’un tableau :

char[] tableau = "Hello".toCharArray();
String chaine = new String(tableau);

Exercice - La médiathèque

On désire modéliser le système de gestion des prêts d’une médiathèque.

Une médiathèque contient des exemplaires de documents. Un exemplaire a un code d’identification propre à la médiathèque. Parmi les documents, on trouve :

  • les livres qui sont définis par un titre, un auteur, un éditeur et un ISBN

  • les revues qui sont définies par un titre, un éditeur et un ISSN

  • les films qui sont définis par un titre, un réalisateur, un distributeur et un ISBN.

Créez les classes Exemplaire, Document, Livre, Revue et Film.

Exercice - La médiathèque - recherche de documents (suite)

Nous allons mettre en place un système de recherche simplifié. Un document doit avoir la méthode boolean correspond(String motCle) qui permet de savoir si un document correspond au mot clé passé en paramètre.

Selon le type de document, la méthode doit retourner true :

  • pour un livre si le titre ou l’auteur ou l’éditeur contient le mot-clé

  • pour une revue si le titre ou l’éditeur contient le mot-clé

  • pour un film si le titre ou le réalisateur ou le distributeur contient le mot-clé

Créez la classe Mediatheque qui contient l’ensemble des exemplaires de la médiathèque. Pour simplifier, les exemplaires et les documents seront codés en dur dans une méthode d’initialisation de la médiathèque.

La classe Médiathèque a la méthode List<Document> rechercher(motCle) qui permet de rechercher une liste de documents qui correspondent au mot clé.

Créez un programme qui permet de saisir un mot-clé et qui retourne la liste des documents correspondant à ce mot-clé.

Exercice - La médiathèque - application d’emprunt (suite)

Les usagers empruntent des exemplaires dans la médiathèque. Un usager a un matricule et un nom. Un usager a également la liste de ses prêts en cours.

Règles pour l’emprunt :

  • Un usager ne peut pas emprunter plus de 6 exemplaires.

  • Un usager ne peut pas emprunter plus de 3 films.

Ajoutez les méthodes emprunter(Exemplaire) et rendre(Exemplaire) a la classe Usager.

Si les règles ne sont pas respectées la méthode emprunter(Exemplaire) doit échouer avec une exception particulière.

Créez un programme qui crée en mémoire un usager à partir d’un matricule et d’un nom saisis par l’utilisateur. Le programme permet ensuite d’effectuer une recherche par mot-clé et de choisir parmi la liste de résultats l’exemplaire que l’on souhaite emprunter.

Le programme doit permettre également d’afficher la liste des prêts de l’usager.

Exercice - La médiathèque - comparaison des documents (suite)

On souhaite pouvoir trier une liste de documents en utilisant la méthode sort qui accepte en paramètre un Comparator. Il doit être possible de trier les documents :

  • suivant leur titre

  • suivant leur type (d’abord les livres, puis les revues puis les films) et ensuite suivant leur titre.

Créer les classes nécessaires implémentant l’interface Comparator et écrivez un programme de test.

Exercice - La médiathèque - date d’emprunt et de retour (suite)

Un exemplaire possède comme propriétés une date d’emprunt et une date de retour lorsqu’il est emprunté. La date de retour est de deux semaines après la date d’emprunt. Attention, si l’emprunt est fait entre le 15 août et la 31 août, la date de retour est le 1er septembre car la médiathèque est fermée à cette période.

Ajoutez les propriétés de date d’emprunt et de date de retour à un Exemplaire. Modifiez le programme d’emprunt pour prendre en compte la date d’emprunt (qui correspond à la date du jour). Un exemplaire déjà emprunté ne peut pas être emprunté à nouveau (utilisez le mécanisme des exceptions pour signaler un problème dans ce cas là).

Astuce

Pour réaliser cet exercice, consultez le chapitre langage_java/les_dates.

Exercice - La médiathèque - chargement depuis un fichier (suite)

On souhaite modifier la classe Mediatheque afin de pouvoir charger le catalogue de la médiathèque depuis un fichier CSV.

Un fichier CSV contient une information concernant un exemplaire pour chaque ligne. Sur une ligne, les colonnes sont séparées par un point-virgule.

Une ligne contient les colonnes suivantes :

  • le code de l’exemplaire

  • une colonne donnant le type de document : L pour un livre, R pour une revue et F pour un film

  • Pour un livre, les colonne suivantes contiennent le titre, l’ISBN, l’auteur et l’éditeur

  • Pour une revue, les colonnes suivantes contiennent le titre, l’ISSN et l’éditeur

  • Pour un film, les colonnes suivantes contiennent le titre, l’ISBN, le réalisateur et le distributeur

S’il existe plusieurs exemplaires pour un même document, la ligne des exemplaires suivant le premier ne contient que le code.

Ci-dessous un exemple de fichier :

1234;L;Le Seigneur des Anneaux;978-2266286268;J.R.R. Tolkien;Pocket
1456
1568
2256;R;Les Cahiers du Cinéma;0008-011X;
6842;F;Alien;B0041KWHU6;Ridley Scott;20th Century Fox
6843
6844

Créez la classe MediathequeLoader capable de créer un objet de type Mediatheque contenant le catalogue chargé depuis un fichier.

Modifiez le programme de recherche d’un document réalisé précédemment afin d’utiliser le MediathequeLoader pour charger le catalogue à partir d’un fichier.

Pensez à gérer les cas d’erreur avec des exceptions spécifiques si le fichier est mal formé (par exemple si le nombre de colonnes n’est pas correct ou si le format des données n’est pas celui attendu).

Astuce

Pour réaliser cet exercice, consultez le chapitre langage_java/entrees_sorties.

Pour découper une ligne selon le séparateur point-virgule, vous pouvez utiliser la méthode split d’une chaîne de caractères.
